Bing::Implementation details

To deal with splash one has to use Eclipse P2 (Povisioning platform) – this is the only way to archive that. The second point is to substitute splash after the first Eclipse restart – for that P2 touchpoint is the only solution. In Eclipse splash can be set in two places:

  1. in configuraiton/config.ini file, as osgi.splashPath
  2. in eclipse.ini file, as -showsplash parameter

The touchpoint plugin is called when Bing's splash feature is being installed. In p2.inf file we have set metaRequirements to our touchpoint, thus making P2 to install the bundle before installing the feature itself. The feature has embedded splash screen that you'll see after restart.

Bing's plugin is activated when you open it's view in Eclipse (see instructions above) and downloads photos of the day from bing.com, converts them to BMP (raw) format—Eclipse supports only this format for splash srceen. Then the plugin figures out Eclipse-feature embracing it and saves BMP-files in the feature's directory. Features unlike ordinary OSGi-bundles are directories, not jar-files.

When one choose new splash, it saves corresponding bmp-file as splash.bmp in Bings-feature directory and next time you will see it as an Eclipse splash.

Bing::Release engineering

The project uses Git-Flow methodology. The building server—Travis-CI—launches building process when git-commit in development branch is tagged on GitHub. The process is described in .travis.yml.
